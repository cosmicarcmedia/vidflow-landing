Tags

Updated July 10, 2026

Tags organize your content library into custom categories: series, campaigns, content pillars, or anything else that fits your brand. They also do routing work, since schedules, auto-import, and Flows can all act on tags.

Tags are available on every plan.

#How it works

A tag is a name and a color scoped to your workspace. Applying a tag just adds its id to that piece of content, so the same tag can sit on text posts, images, clips, and episodes at once. Nothing else in Vidflow reads the tag's color, only its name and id, so color is purely a visual aid for you and your team.

#Create a tag

  1. Open the Content Library and click Manage Tags in the filter row.
  2. Under New Tag, type a name and pick a color. Ten preset swatches are offered, or click the color square for a custom color.
  3. Click Add.

#Edit or delete a tag

In the same Manage Tags dialog, the Existing Tags list shows every tag in the workspace. Use the controls on a tag's row to rename it, change its color, or delete it.

Deleting asks for confirmation and removes the tag from all content that carries it. The content itself is not deleted.

#Tag your content

  • One item: open it and click the tag pills in the editor. See Edit a post.
  • Many items: use Re-Tag Content in Bulk Manage. See Bulk actions.
  • Episodes: the episode's Details tab has a Tags field. Separate tags with commas.
  • Automatically: auto-import sources and Flows can tag content for you (see below).

#Where tags work

  • Library filters: narrow the library to one tag from the sort and filter menu. See Content Library basics.
  • Schedules: a schedule can be fed by tags, so only content carrying any of those tags enters that queue. See Schedules and queues.
  • Auto-import: a source can tag everything it brings in. See Auto-import from Google Drive or Dropbox.
  • Flows: the Tag Content node applies tags during an automated run. See Flows overview.

#Get better results

  • Design a small, stable tag set before you scale up content, for example one tag per content pillar or per recurring series. A handful of tags you reuse consistently routes and filters far better than dozens of one-off tags.
  • Give each tag a distinct color if you rely on the library's grid view to scan visually; a shared or default color makes tags harder to tell apart at a glance.
  • Put the Tag Content node right before a Flow's Save node. Anything saved without passing through a Tag Content node has no tags, so it will not match tag-based schedules or filters later.
  • Use consistent tag names across auto-import sources and Flows if you want one schedule to catch content from both. Routing matches on the tag itself, not on which feature applied it.

#Troubleshooting

  • Deleting a tag removed it from my content by mistake. This is expected: deleting a tag strips it from everything that carried it, though the content itself is untouched. Recreate the tag and re-tag the content with Bulk Manage if needed.
  • A schedule fed by a tag isn't picking up new content. Confirm the new content actually carries that tag; auto-import and Flows only tag content when you configure them to.
  • The tag list is empty in Manage Tags. Tags are scoped to the current workspace. If you expect tags from another brand, switch workspaces first.
