The Native clip engine

Updated July 9, 2026

Native is Vidflow's in-house clip engine. It selects moments from the transcript, reframes the video, burns in captions, and renders the clips on Vidflow's own infrastructure. There is no third-party key to connect.

Beta note

Beta note: The Native engine is in beta. It is the default engine in the Clip Generator.

#What you need

Native needs the actual video file uploaded to the episode. It does not work from a link-only source.

Important

Important: If your episode was added as a YouTube link with no uploaded media, Native shows this message: "Native clips need the actual video file. This episode is a link (for example YouTube) with no uploaded media." Upload the video file for this episode, or switch to the OpusClip engine, which can fetch link-only sources itself.

#Options

From the Clip Generator, select the Native tab, then set:

  • Number of clips: a number from 1 to 20. Default is 8.
  • Clip length: Any length (default), Under 1 min, 1 to 3 min, or 3 to 5 min.
  • Caption style: Standard (house default), Box Background, Bold Center Pop, Karaoke Band, Clean Minimal, Hype Impact, or Editorial Serif.

Click Generate Clips to start the run.

#How it works

Native runs in two stages. First, a selection pass reads the transcript and picks brand-aware moments. Then each selected moment goes through an automatic render job: reframing to vertical, captions burned in using your chosen style, and a thumbnail pulled from about a third of the way into the clip.

Finished clips appear as clip items in your Content Library, tagged with a Native source, and count toward your workspace's clip generation allowance. For plan limits, see Plans and features and Usage limits and monthly allowances.

#Troubleshooting

If Native tells you it needs a video file, either upload the file to that episode or use OpusClip instead. For stuck or missing clips after generating, see Clips troubleshooting.
