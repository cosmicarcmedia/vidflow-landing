Help / Flows automation

Flow nodes reference

Updated July 10, 2026

Every node you can place in a flow, with its options. Nodes are listed in the same groups as the editor palette: Triggers, Processing, Publishing, Logic, and Reviews.

#How it works

Each node is a small, self-contained step: a trigger starts a run, a processing node calls the AI or clip engine and produces one or more in-memory drafts, a logic node changes what happens next, and a publishing node is what actually commits a draft to your library or a queue. A run walks the graph from its trigger, executing each connected node once and passing its drafts downstream to whatever it feeds into. The same node type behaves identically whether you place it in a flow you built yourself or find it inside the First Content Pack template every workspace starts with.

Two things apply to every node:

  • Each node has a Label field in its configuration panel so you can rename it on the canvas.
  • Generated content travels down the flow as drafts. Enrichment nodes (captions, images, tags) modify those drafts, but nothing is stored until the drafts reach a Save or Post node.

#Triggers

A flow has one trigger, chosen in the toolbar. The trigger nodes in the palette match the same three options.

Episode Upload

Starts the flow whenever a new episode is created in this workspace. No configuration. The flow's Active switch must be on for it to fire, and each run waits for the episode's transcript before starting; see Runs, history, and troubleshooting.

Google Drive File

Starts the flow from a file dropped into a Google Drive folder.

  • Folder ID: the Drive folder to watch.
Beta note

Beta note: The Google Drive trigger is in beta. Its folder watcher is not wired up yet, so these flows do not fire on their own. You can still build and save one, and run it manually.

Manual Trigger

Runs on demand. When you start the run you pick an episode (uses its transcript) or a saved script (uses the script text) as the source. No configuration.

#Processing

Generate Clips

Generates short clips from the source episode. Pick the Engine first; the options change with it.

  • Engine: OpusClip or Native (Beta).

With Native (Beta) (Vidflow's own moment selection and rendering, no external account needed):

  • Clip count: 1 to 20.
  • Duration band: 0-1m, 1-3m, 3-5m, or any.
  • Caption style: Standard (house default), Bold Center Pop, Karaoke Band, Clean Minimal, Hype Impact, or Editorial Serif.
  • Genre (optional) and Topic / prompt (optional) to steer moment selection.
  • Auto-approve top clips: on by default.

With OpusClip (requires a connected OpusClip account, see Using OpusClip with Vidflow):

  • Mode: ClipBasic or ClipAnything.
  • Aspect Ratio: portrait, landscape, or square.
  • Clip lengths: any mix of Auto, <30s, 30-60s, 60-90s, and 90s to 3min.
  • In ClipAnything mode, What should Opus clip? takes a free-text prompt. In ClipBasic mode you get Genre (optional) and Keywords (comma-separated) instead.
  • Remove filler words: off by default.

Magic Ideas

Generates content ideas from the episode for downstream editorial work.

  • Idea Type: clips, posts, themes, or mixed.
  • Tone: professional, casual, humorous, or educational.
  • Max Ideas: how many to generate.

Magic Text Posts

Writes AI text posts from the episode transcript.

  • Post types: any mix of Quote, Insight, Tip, Story, Question, and Statistic.
  • Tone: Professional, Casual, Witty, Inspirational, or Educational.
  • Platforms: X, LinkedIn, Instagram, Facebook.
  • Posts per type: 1 to 10.
  • When done: needs_approval or auto_approve, which sets the approval state the posts land with.
  • Include Hashtags and Include Emojis: both on by default.
  • Custom Prompt: extra instructions for the writer.

Blog Posts

Generates long-form blog content from the episode.

  • Post Type: Full Article, Summary, Key Takeaways, or How-To.
  • Length: Short, Medium, or Long.
  • Include SEO and Include Call-to-Action: both on by default.
  • Custom Prompt: extra instructions.

Script Generator

Turns episode context into a script.

  • Script Type: Video Script, Podcast Intro, Sales Pitch, Educational, or Storytelling.
  • Duration: 30s, 60s, 2min, 5min, or Custom.
  • Tone: Professional, Casual, Energetic, Calm, or Humorous.
  • Include Timestamps: on by default.
  • Custom Prompt: extra instructions.

Generate Captions

Writes a tailored caption for each selected platform on every piece of content produced upstream.

  • Platforms: X, LinkedIn, Instagram, Facebook.
  • Tone: Professional, Casual, Witty, Inspirational, or Educational.

Generate Images

Turns concepts or upstream posts into brand-styled images.

  • Preset: appears when your workspace has image presets; applying one fills the prompt. Presets are managed in Brand Settings under Image Presets.
  • Image type: Quote Card, Infographic, Carousel Slide, Thumbnail, Announcement, or General.
  • Image concepts from: episode (Vidflow writes distinct concepts from the episode, one image each) or upstream (one image per upstream idea or post, for example from a Magic Ideas node connected before this one).
  • Number of images (or Number of slides in carousel mode): 1 to 10. Shown when concepts come from the episode.
  • Aspect Ratio: 1:1, 4:5, 3:4, or 1.91:1.
  • Style (optional) and Prompt / direction (optional): free-text steering.
  • When done: needs_approval or auto_approve.
  • Combine into one carousel: generates one coherent carousel saved as a single post instead of separate images.
  • Caption per platform (optional): pick platforms to get a tailored caption per platform on each image, with no separate Generate Captions node needed. With none selected, each image gets one base caption.

Tag Content

Applies workspace tags to every piece of content moving through this point in the flow.

  • Tags: pick from the workspace's existing tags. Tags are created in the Content Library first; see Tags.

Place this node right before a Save node so everything being saved picks up the tags.

#Publishing

Save to Library

Commits everything assembled upstream (posts, images, captions, tags) into the Content Library as drafts. No options. Until a Save node runs, content stays in limbo and is not saved anywhere, so every producing flow needs this node or one of the other publishing nodes at the end.

Save to Queue

Saves upstream content to the library and adds it to a publishing queue.

  • Queue: pick an existing queue or choose + Create new queue… to make one inline.
  • Priority: Low, Normal, or High.
  • Add to Calendar: off by default; turning it on reveals a Scheduled Date field.

Schedule Post

Saves upstream content and schedules it.

  • Platforms: X, LinkedIn, Instagram, Facebook.
  • Time (UTC): defaults to 14:00.
  • Repeat: Once, Daily, Weekdays, or Weekly.
  • Delay Hours: pushes the first slot out from the time of the run.

Post to X, Post to Instagram, Post to LinkedIn

Each of these saves the upstream content and queues it for that platform. No options beyond the label. Publishing then follows your normal queue and connected-account setup; see Schedules and queues.

#Logic

Router

Splits the flow into parallel branches. Every connected branch runs. Drag a node close to the router to attach it as a branch; to detach one, right-click the connected node and choose Disconnect, or right-click the connection line and choose Delete Connection.

Routers have no configuration of their own. To control what travels down each branch, put a filter on the branch's connection; see the branch filters section of Build your first flow.

Delay

Offsets publishing timing for everything downstream.

  • Duration plus Unit: minutes, hours, or days.

The delay pushes out when downstream content is scheduled or queued. The run itself does not pause.

Condition

Evaluates rules before continuing. If they pass, the flow continues; if not, this branch stops.

  • Conditions (one per line, all must pass): each line is written as field operator value, for example title contains AI or status == ready. Fields: title, status, description, tags, postType, kind, triggerType. Operators: ==, !=, contains, startsWith.

#Reviews

Approvals

Routes upstream content through an approval step before it moves on. See Client approvals overview for how approvals work in Vidflow.

  • Approval Type: Auto-Approve, Client Review, Internal Review, or Both.
  • Notify On: any mix of Approval Needed, Approved, and Rejected.
  • Require Comment: off by default.
  • Timeout Days: 1 to 30.

Notify Owner

Sends an in-app notification when the run reaches this node.

  • Message Template: the notification text. Left empty, it sends a default step-completed message naming the episode.

#Get better results

  • On Generate Clips, prefer the Native engine when you want clips with no external account and full control over duration and caption style; reach for OpusClip when you specifically want its ClipAnything free-text targeting or you already have an OpusClip account connected.
  • On Magic Text Posts, Generate Captions, and Blog Posts, use the Custom Prompt field to lock in structure you always want (a CTA format, a disclaimer, a preferred opening line) instead of relying on the tone dropdown alone.
  • On Generate Images, feed it upstream posts (from Magic Text Posts or Magic Ideas) rather than leaving it as the first node in a branch. Attached-to-post images inherit that post's angle, which reads as more intentional than a standalone generic graphic.
  • On Tag Content, always place it upstream of every Save or Post node in the flow, not just one branch, if you want consistent tags across everything the flow produces.
  • On Condition, write one field-operator-value expression per line rather than trying to cram logic into a single line; each line is AND-ed automatically, so this reads clearer and is easier to fix later.

#Troubleshooting

  • A node's step shows "skipped" in the run log. Most often the node had nothing upstream to act on, or a Condition on that branch did not pass. See Runs, history, and troubleshooting for the full list of skip reasons.
  • Generate Clips did nothing. Confirm the run has an episode in context; without a trigger episode or an Episode ID for Run, clip and other generation nodes are skipped.
  • A node type is not listed here. It is not executable yet and is marked beta in the editor; placing it is safe, but the run will skip it with "Node type is not executable yet."
