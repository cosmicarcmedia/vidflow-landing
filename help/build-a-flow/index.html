Build your first flow

Updated July 10, 2026

Build a flow that turns each new episode into content automatically. This walkthrough covers the editor, connecting nodes, router branches, branch filters, and the first run.

Note

Note: Building and running a flow you create yourself requires the Pro or Agency plan. Every workspace already has one flow running for free: First Content Pack, installed automatically and active from day one. See Flows overview and Plans and features.

#How it works

A flow is a graph of nodes saved as one record: a trigger, the processing and publishing nodes you connect to it, and the edges between them. Running a flow walks that graph starting from the trigger, executes each node in turn (an AI node calls the model, a Save node writes to the library, and so on), and writes a step-by-step log you can read afterward in History. Nothing is committed to your library until a run reaches a Save or Post node, so you can safely build and test a flow before it ever creates real content.

#Create the flow

  1. Open Flows in the sidebar.
  2. Click New Flow, or click Use Template on a template (including First Content Pack, if you want to start from the flow every workspace already has) to start from a prebuilt pipeline.
  3. In the left panel, enter a Workflow Name and an optional Description.
  4. In the toolbar, pick the trigger: Episode Upload, Google Drive File, or Manual Trigger.

#Get to know the editor

The left panel holds the node palette, grouped into Triggers, Processing, Publishing, Logic, and Reviews. The toolbar has the trigger picker, the Active switch, an Episode ID for Run field, undo and redo, the node legend, Snap, Tidy up, zoom controls with a recenter button, Run, and Save.

A few useful controls:

  • Drag the empty canvas to pan, and scroll to zoom.
  • Snap aligns nodes to the grid while you drag. Tidy up auto-arranges the whole flow into clean columns.
  • The question mark button opens the Node Type Legend, a searchable catalog of every node. Clicking a node there adds it to the canvas.
  • Keyboard shortcuts: Cmd+S (Ctrl+S on Windows) saves, Cmd+Z undoes, Cmd+Y redoes, Delete removes the selected node or connection, and G toggles the grid.

#Add and connect nodes

  1. Drag a node from the palette onto the canvas, or click it to place it.
  2. Drop the node near an existing node's output and it connects automatically. Drop it onto an existing connection and it splices in between the two nodes.
  3. To connect manually, click the small port on the right edge of a node, then click the node it should feed into.
  4. Click a node to open its configuration panel. Every node has a Label field so you can rename it; the other options are listed in the Flow nodes reference.

Right-click a node for Edit, Duplicate, Disconnect (removes all of that node's connections), and Delete. Right-click a connection for Delete Connection.

#Router branches

A Router splits the flow into parallel branches, and every connected branch runs. To add a branch, drag a node close to the router and it connects on its own.

To remove a branch, right-click the connected node and choose Disconnect. You can also right-click the connection line itself and choose Delete Connection to remove only that link. Moving a node around the canvas never detaches it; removal is always an explicit action.

#Branch filters

Any connection can carry a filter, so only matching content continues down that branch. Click the filter icon in the middle of a connection to open the Branch filter panel, then click + Add condition.

Each condition is a field, an operator, and a value:

  • Fields: Content type, Title / name, Status, Tags, Trigger.
  • Operators: is, is not, contains, starts with, ends with.
  • Content type values: Clip, Image, Text post, Video, Blog / article, Carousel, Episode.

All conditions on a connection must pass. A connection with no conditions always passes. This is how one router can send clips to one queue and text posts to another.

#Save and run

  1. Click Save (or press Cmd+S). A flow must be saved before it can run; Run stays disabled in a new, unsaved flow.
  2. Run it from the Flows page with Run: pick an episode (uses its transcript) or a saved script (uses the script text), then click Run flow.
  3. Or run from the editor: enter an episode's ID in Episode ID for Run, then click Run. Without an ID, the run has no source and generation steps are skipped.
  4. For an episode-upload flow, switch Active on so it fires whenever a new episode is created.

If you navigate away with unsaved edits, the editor asks Leave with unsaved changes? with the choices Stay, Discard, and Save & Leave.

#Get better results

  • Test with Episode ID for Run against a real, already-transcribed episode while you build. That way you see actual generated content in each step's log instead of guessing whether a node is configured correctly.
  • Use Tidy up once your flow has more than five or six nodes. A cleanly laid out canvas makes it much faster to spot a missing connection or a branch that dead-ends without a Save node.
  • Name every node you touch through its Label field, especially on a flow with more than one Generate Images or Generate Captions node. The run history shows node labels, and "Generate Captions" repeated three times with no labels is much harder to debug than "Captions - IG/X" and "Captions - LinkedIn."
  • Put a Tag Content node right before every Save node, not just one of them, if you want consistent tagging across every branch of a router.
  • Keep branch filter conditions simple, one field and one operator per line. Complex conditions are harder to verify are doing what you think, and each line is already AND-ed together if you need more than one.

#Troubleshooting

  • Run is disabled in the editor. The flow has never been saved. Click Save first.
  • Nothing appeared in the library after a run. The flow has no Save node, so generated content was discarded at the end of the run. End the flow in Save to Library, Save to Queue, or a Post node.
  • The flow never fires on upload. Check that the trigger is Episode Upload and the Active switch is on. Then see Runs, history, and troubleshooting for the transcript wait.
