Create and share an approval link

Updated July 10, 2026

Create a link that bundles content for a client to review, then copy it and send it. This article covers creating, sharing, closing, and reopening links.

#How it works

Creating a link snapshots the content you selected into a batch and generates a random share token, the URL your client opens is just that token. The link carries no login of its own; opening it is enough to see and act on everything in the batch. Approving an item on the review page updates that item's approval status immediately and directly, so the moment your client clicks Approve, it's reflected in your Content Library and to any schedule checking that item's approval state, with no separate sync step.

#Before you start

  • Approvals are available on paid plans, Starter and up. On the Free plan the Create Approval Link button is locked and prompts you to upgrade. See Plans and features.
  • The content you want reviewed needs to be in your Content Library first.

#Create the link

  1. In the app rail, open Approvals (in the Publish section).
  2. Click Create Approval Link. If this is your first link, the page shows a Create First Link button instead.
  3. Enter a Link Title, for example "Week 12 Content Review". The client sees this title when they open the link.
  4. Optionally set Expires In (days) to a number from 1 to 90. Leave it blank and the link never expires.
  5. Under Select Content, open the picker and select the items the client should review.
  6. Click Create Link.

#Share it

The dialog switches to Approval Link Created and shows the URL in a Shareable Link field.

  1. Click Copy to put the URL on your clipboard.
  2. Click Preview Link to open the review page yourself and check it before sending.
  3. Send the URL to your client by email, text, or any other channel. It is a plain URL; nothing else is required.
  4. Click Done to close the dialog.
Important

Important: Anyone who has the URL can open it, approve, and comment. There is no login gate. Share it only with the people who should sign off.

#Manage existing links

Every link you have created appears on the Approvals page with a status badge: Open, Closed, or Expired. Each card shows the creation date, the expiry date if one is set, and the share URL with Copy and Open buttons.

  • Close a link to shut it off. Clients who open it see an invalid-link message.
  • Reopen a closed link to turn it back on. The same URL starts working again, so there is nothing new to resend.
  • Links cannot be edited after creation. Title, expiry, and content are fixed. For a different set of content, create a new link.
Note

Note: Reopening does not revive an expired link. Once the expiry date passes, the card shows Expired and the link stays dead. Create a new link instead.

#Get better results

  • Select content right before you send, not days ahead. The picker snapshots whatever you select at creation time; anything added to the library afterward isn't automatically included. If more content lands after you create a link, either add a fresh batch to a new link or hold off creating the link until everything is ready.
  • Write titles your client will recognize without context, like "Week 12 Content Review" or a campaign name, since the title is the first and sometimes only thing they see before opening the content.
  • Close links you're done with instead of leaving them open indefinitely. A closed link can't be used to approve anything new, which keeps old, already-actioned links from becoming a confusing second copy of your current batch.
  • Preview every link yourself before sending it. Click Preview Link to see exactly what your client will see, catch anything mis-selected, and confirm your branding looks right, before it ever reaches them.

#Troubleshooting

Your client reports "Approval link not found":

  1. Check the link's badge on the Approvals page. If it is Closed, click Reopen.
  2. If it is Expired, create a new link with the same content.
  3. If it shows Open, the URL probably got truncated in transit. Click Copy on the link card and resend the full URL.
