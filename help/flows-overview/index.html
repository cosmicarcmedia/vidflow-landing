Flows overview

Updated July 10, 2026

Flows are Vidflow's automation builder. You draw a pipeline once, and every new episode comes out the other side as clips, text posts, and images, saved to your library and queued for publishing.

#What a flow is

A flow is a chain of nodes on a canvas. It starts with a trigger, runs your content through processing nodes (clips, text posts, blog posts, captions, images, tags), and ends in a publishing node that saves the results. Until a Save node runs, nothing generated by the flow is stored, so every flow should end in Save to Library, Save to Queue, or another publishing node.

Flows belong to a workspace. Open Flows in the sidebar to see the flows for the workspace you are in.

Note

Note: Building, editing, and manually running a flow in the Flows editor requires the Pro or Agency plan. The requirement is enforced when you create a flow and when you run one. See Plans and features. The one exception is the First Content Pack below, which runs automatically on every plan.

#Your first flow runs itself

Every new workspace gets a flow named First Content Pack installed automatically, no setup required. It is active by default and fires the moment you add your first episode: it generates a handful of clips, a few text posts for Instagram, X, and LinkedIn, and a small set of quote-card images, writes per-platform captions on all of it, tags everything "First pack," and drops the results into your default publishing queue.

This runs on every plan, including Free, because it is how a new workspace gets its first real content without anyone having to learn the flow editor first. You can watch it build on the onboarding screen after you add your first episode; it usually finishes within a few minutes. The flow itself is a normal flow, so once you're on Pro or Agency you can open it in the editor and change what it generates.

#Triggers

  • Episode Upload starts the flow whenever a new episode is created in the workspace. The flow's Active switch must be on. These runs wait for the episode transcript to finish before they start; see Runs, history, and troubleshooting.
  • Manual Trigger runs on demand. When you click Run, you pick an episode or a saved script as the source.
  • Google Drive File starts a flow from a file dropped in a Drive folder.
Beta note

Beta note: Episode-upload and manual triggers are fully live. The Google Drive trigger is in beta: its folder watcher is not wired up yet, so Drive flows do not fire on their own (you can still run them manually). Webhook and script triggers are also in beta and are not available in the editor yet.

#What runs live

Every processing and publishing node runs for real: Clips, Magic Text Posts, Blog Posts, Magic Ideas, Scripts, Captions, Generate Images, Save to Library, Save to Queue, Schedule, Post to X, Instagram, and LinkedIn (queued), Approvals, Delay, Condition, Notify Owner, and Router branching. The full list of nodes and their options is in the Flow nodes reference.

Generation nodes draw from your plan's monthly allowances. See Usage limits and monthly allowances.

#Managing your flows

Each card on the Flows page shows a wireframe preview of the flow, its trigger badge, and an Active switch. Use Run to start it, Edit to open the editor, and the card menu for History, Disable or Enable, and Delete. A violet Running badge means a run is in progress; an amber Waiting badge means a run is held until the episode's transcript is ready.

Important

Important: Deleting a flow also removes its run history. The app confirms before deleting.

#Templates

The Templates row seeds the editor with a prebuilt flow you can adjust and save. Podcast Support turns each uploaded episode into short clips and drops them into a publishing queue. Full Stack fans an upload out into clips, X-ready text posts, and generated images, then queues everything. First Content Pack is also available here as a starting point if you want to build a second flow from it.

#Get better results

  • Start from a template or from First Content Pack rather than a blank canvas. Adjusting an existing pipeline is faster than wiring processing and publishing nodes from scratch, and it guarantees you end in a working Save or Post node.
  • Keep one flow doing one job. A flow that tries to handle every content type at once gets hard to read and hard to fix; a Router with a couple of focused branches is easier to reason about than a dozen parallel nodes off one trigger.
  • Turn a flow's Active switch off while you are actively editing it, then back on once you're happy, so a half-finished pipeline does not fire on the next episode upload.
  • Check History after the first few runs of any new flow, not just when something looks wrong. Skipped and failed steps show up there even when the flow "completed."

#Troubleshooting

  • Flows is greyed out or shows an upgrade prompt. Manually building and running flows needs the Pro or Agency plan. The First Content Pack still runs for you automatically regardless of plan. See Plans and features.
  • A flow never fires on new episodes. Confirm its trigger is Episode Upload and its Active switch is on. See Runs, history, and troubleshooting.
  • Nothing showed up in the library after a run. The flow has no Save node, so everything it generated was discarded at the end of the run. See Build your first flow.
  • My first episode didn't get a content pack. Check Flows for a flow named "First Content Pack" and confirm it is Active. If it was deleted or disabled, recreate it from the First Content Pack template.
