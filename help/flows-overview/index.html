Help / Flows automation

Flows overview

Updated July 9, 2026

Flows are Vidflow's automation builder. You draw a pipeline once, and every new episode comes out the other side as clips, text posts, and images, saved to your library and queued for publishing.

#What a flow is

A flow is a chain of nodes on a canvas. It starts with a trigger, runs your content through processing nodes (clips, text posts, blog posts, captions, images, tags), and ends in a publishing node that saves the results. Until a Save node runs, nothing generated by the flow is stored, so every flow should end in Save to Library, Save to Queue, or another publishing node.

Flows belong to a workspace. Open Flows in the sidebar to see the flows for the workspace you are in.

Note

Note: Flows require the Pro or Agency plan. See Plans and features. The requirement is enforced when you create a flow and when you run one.

#Triggers

  • Episode Upload starts the flow whenever a new episode is created in the workspace. The flow's Active switch must be on. These runs wait for the episode transcript to finish before they start; see Runs, history, and troubleshooting.
  • Manual Trigger runs on demand. When you click Run, you pick an episode or a saved script as the source.
  • Google Drive File starts a flow from a file dropped in a Drive folder.
Beta note

Beta note: Episode-upload and manual triggers are fully live. The Google Drive trigger is in beta: its folder watcher is not wired up yet, so Drive flows do not fire on their own (you can still run them manually). Webhook and script triggers are also in beta and are not available in the editor yet.

#What runs live

Every processing and publishing node runs for real: Clips, Magic Text Posts, Blog Posts, Magic Ideas, Scripts, Captions, Generate Images, Save to Library, Save to Queue, Schedule, Post to X, Instagram, and LinkedIn (queued), Approvals, Delay, Condition, Notify Owner, and Router branching. The full list of nodes and their options is in the Flow nodes reference.

Generation nodes draw from your plan's monthly allowances. See Usage limits and monthly allowances.

#Managing your flows

Each card on the Flows page shows a wireframe preview of the flow, its trigger badge, and an Active switch. Use Run to start it, Edit to open the editor, and the card menu for History, Disable or Enable, and Delete. A violet Running badge means a run is in progress; an amber Waiting badge means a run is held until the episode's transcript is ready.

Important

Important: Deleting a flow also removes its run history. The app confirms before deleting.

#Templates

The Templates row seeds the editor with a prebuilt flow you can adjust and save. Podcast Support turns each uploaded episode into short clips and drops them into a publishing queue. Full Stack fans an upload out into clips, X-ready text posts, and generated images, then queues everything.
